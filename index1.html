Network Layer – Overview
🔵 Definition

The Network Layer is Layer 3 of the OSI model. It is responsible for moving data packets from the source computer to the destination computer across multiple networks, using routers and IP addresses.

🔴 Key Points
  • 3 Functionalities: Forwarding, Routing, Connection Setup
  • 4 Services: Guaranteed Delivery, Bounded Delay, In-order Delivery, Max Jitter
  • Topics: Router internals, IP, IPv4/IPv6, NAT, Fragmentation, ICMP
  • Network Layer = Layer 3 = Uses IP Addresses
🟢 Simple Analogy

Think of the Network Layer as the postal system of the internet — just like the post office decides the route for a letter, the Network Layer decides the route for data packets.

⚡

Quick Summary: Network Layer = Layer 3. It handles forwarding (local), routing (global path), and connection setup using routers.

2
Forwarding
🔵 Definition

Forwarding is the process of moving a packet from the incoming link to the correct outgoing link within a single router, using its forwarding table.

🔴 Key Points
  • Every router has a Forwarding Table
  • Router reads the header value of the arriving packet
  • It uses that value to look up the forwarding table
  • The table gives the output link interface number
  • Forwarding is a local, per-router action
  • Routing algorithm builds the forwarding table
📊 How Forwarding Works
1

Routing Algorithm runs → Creates the Local Forwarding Table

↓
2

Packet arrives with header value 0111

↓
3

Router looks up table: 0111 → Output Link 2

↓
4

Packet is sent out through Output Link 2 ✅

Table: 0100→Link3 | 0101→Link2 | 0111→Link2 | 1001→Link1

🟢 Real-Life Example

Hotel reception: Guest (packet) arrives → receptionist (router) checks room number (header) in register (forwarding table) → directs to correct floor (output link).

⚡

Quick Summary: Forwarding = local action at ONE router. Uses forwarding table. Header value → output link.

3
Routing
🔵 Definition

Routing is the process of finding the best path for packets to travel from source to destination, involving all routers in the network communicating via routing protocols.

🔴 Key Points
  • Routing involves ALL routers in the network (not just one)
  • Routers talk to each other via Routing Protocols
  • Routing Algorithms can be: Centralized or Decentralized
  • L2 Switches → forward based on MAC addresses (Link Layer)
  • L3 Routers → forward based on IP addresses (Network Layer)
  • Routers do NOT run Application or Transport layer protocols
FeatureForwardingRouting
ScopeSingle RouterEntire Network
SpeedVery Fast (Hardware)Slower (Software)
FunctionMove packet to outputCalculate best path
🟢 Real-Life Example

Routing = Google Maps planning entire route Chennai→Mumbai. Forwarding = Each traffic signal directing you left/right at that intersection.

⚡

Quick Summary: Routing = global path planning by all routers. Forwarding = local action at one router. L2 uses MAC, L3 uses IP.

4
Connection Setup & Network Service Models
🔵 Definition

Connection Setup is a process where routers along the chosen path must exchange "handshake" messages to prepare state before any data packets can flow.

🔴 Key Points
  • Only some network architectures require connection setup
  • Routers along path must handshake before data flows
  • NSM defines characteristics of end-to-end packet transport
  • NSM answers: reliable delivery? in-order? timing? congestion feedback?
🟢 Real-Life Example

Connection setup = booking a train ticket before travelling — you first reserve seat (setup), confirm path (route), then train runs (data transfer). No booking = datagram style.

⚡

Quick Summary: Connection setup = pre-data handshake. NSM defines delivery guarantees, ordering, timing, congestion feedback.

5
Network Service Model – 5 Types
🔵 Definition

The Network Service Model defines the promises of end-to-end packet delivery between sending and receiving end systems.

🔴 All 5 Services
  • 1. Guaranteed Delivery – Packet WILL reach destination (no loss)
  • 2. Guaranteed Delivery with Bounded Delay – Within a fixed time (e.g., 100ms)
  • 3. In-Order Packet Delivery – Arrive in same order as sent
  • 4. Guaranteed Minimal Bandwidth – Minimum data rate guaranteed
  • 5. Security Services – Source encrypts, destination decrypts
  • Jitter = variation in time between successive packet arrivals
🟢 Real-Life Examples
  • Guaranteed Delivery = Registered post
  • Bounded Delay = Express courier (within 24 hrs)
  • In-Order = Numbered parcel delivery
  • Jitter control = Smooth video call (no stuttering)
⚡

Quick Summary: 5 services: Delivery, Bounded Delay, In-order, Min Bandwidth, Security. Jitter = variation in packet arrival time.

Virtual-Circuit vs Datagram Networks
🔵 Definition

Virtual-Circuit (VC) = Connection-oriented; path set up before data flows. Datagram = Connectionless; each packet sent independently with no prior setup.

🔴 Key Points
  • Connection service at Network Layer → Virtual Circuit (VC)
  • Connectionless service at Network Layer → Datagram
  • Examples of VC: ATM, Frame Relay
  • Example of Datagram: Internet (IP)
FeatureVirtual CircuitDatagram
ConnectionRequired (setup)Not required
PathFixed for all packetsDifferent per packet
OrderingIn orderMay be out of order
ExampleATM, Frame RelayInternet (IP)
🟢 Analogy

VC = Train with reserved seats — fixed path. Datagram = Letters by post — each may go through different post offices.

⚡

Quick Summary: VC = connection-oriented, fixed path. Datagram = connectionless. Internet uses Datagram.

8
Virtual Circuit – Structure & Working
🔵 Definition

A Virtual Circuit (VC) is a logical path where a fixed route is established from source to destination, and all packets follow that same path using VC numbers assigned per link.

🔴 Key Points
  • A VC consists of 3 things: (1) Path, (2) VC numbers, (3) Forwarding table entries
  • Each packet carries a VC number in its header
  • VC number can be different on each link
  • Each router replaces old VC number with new one from its forwarding table
  • Examples: ATM and Frame Relay
🟢 Analogy

Relay race: Each runner (link) carries a baton (VC number). At each handoff, baton gets re-tagged with new number for that leg. Identity continues but tag changes at each hop.

⚡

Quick Summary: VC = path + VC numbers + forwarding table entries. VC number changes (translated) at every router. ATM and Frame Relay use VC.

10
Why Different VC Numbers on Each Link?
🔵 Definition

VC numbers change per-link for efficiency, independence, and reduced coordination overhead.

🔴 3 Reasons
  • Reason 1 – Smaller Header: Per-link numbers are locally scoped (smaller) → fewer bits needed in header
  • Reason 2 – Independence: Each link chooses own VC number independently without coordination
  • Reason 3 – No Coordination Overhead: Global VC number would require all routers to agree → very inefficient
  • New VC → entry ADDED to forwarding table
  • VC terminates → entries REMOVED from all forwarding tables on path
⚡

Quick Summary: Per-link VC numbers = smaller header + independent + no coordination. New VC → add entry. VC ends → remove entry.

11
Virtual Circuit – 3 Phases
🔵 Definition

Every Virtual Circuit goes through 3 phases: Setup, Data Transfer, and Teardown. Routers must maintain connection state during all phases.

📊 3 Phases Flow
1

VC Setup: Transport layer → Network layer → specifies receiver → Network layer finds path, assigns VC numbers, adds forwarding table entries, may reserve bandwidth

↓
2

Data Transfer: Packets flow along established VC path with VC numbers

↓
3

VC Teardown: Sender/receiver signals end → Network layer removes forwarding table entries at all routers on path

🔴 Key Points
  • VC routers must maintain connection state information
  • Signaling Messages = messages to initiate/terminate a VC
  • Protocols governing these = Signaling Protocols
🟢 Analogy

Phase 1 = Dialing phone (network finds path). Phase 2 = Talking. Phase 3 = Hanging up (resources freed).

⚡

Quick Summary: VC has 3 phases: Setup → Data Transfer → Teardown. Signaling messages used for setup/teardown. Routers store connection state.

12
Datagram Network
🔵 Definition

A Datagram Network is connectionless — each packet carries the destination address and is forwarded independently by routers using their forwarding tables. No connection setup needed.

🔴 Key Points
  • No connection setup — just stamp address and send
  • Each router uses destination address to look up forwarding table
  • IPv4 addresses are 32 bits = 2³² = over 4 billion possible addresses
  • Solution: Routers use Address Prefix Matching (not exact address)
🟢 Analogy

Letter by post: Write destination address → drop in mailbox → each post office (router) independently decides the next hop. No path booked in advance.

⚡

Quick Summary: Datagram = connectionless, no setup. Each packet routed independently. IPv4 = 32 bits = 4 billion addresses.

13
Longest Prefix Matching
🔵 Definition

Longest Prefix Matching: when a destination address matches multiple entries in the forwarding table, the router picks the entry with the longest (most specific) matching prefix.

🔴 Key Points
  • Routers store address prefixes, not full addresses
  • Longest Prefix = Most Specific Match = Always Chosen
  • "Otherwise" entry = Default route for all unmatched addresses
📐 Numerical Problem

Forwarding Table:

Prefix: 11001000 00010111 00010 → Link 0 Prefix: 11001000 00010111 00011000 → Link 1 Prefix: 11001000 00010111 00011 → Link 2 otherwise → Link 3

Address 2: 11001000 00010111 00011000 10101010

Matches: 11001000 00010111 00011000 (24 bits) → Link 1 Also matches: 11001000 00010111 00011 (21 bits) → Link 2 Longest match = 24 bits → Output: Link Interface 1 ✅

Q: Address = 11001000 00010111 00100000 00000001 → Which link?

Answer: No prefix matches → Default "otherwise" → Link 3 ✅

⚡

Quick Summary: Multiple prefixes match → choose LONGEST. "Otherwise" = default route (Link 3).

14
VC vs Datagram – Forwarding Table Update Speed
🔵 Definition

VC forwarding tables update every time a connection is set up or torn down (microseconds). Datagram tables update slowly via routing protocols (every 1–5 minutes).

🔴 Key Points
  • Datagram: table updates every 1–5 minutes (slow)
  • VC: table updates at microsecond speed (fast)
  • Datagram: tables can change anytime → packets may take different paths → may arrive out of order
  • VC: all packets follow fixed path → always in order
⚡

Quick Summary: VC table updates = microseconds. Datagram table = every 1–5 minutes. Datagram packets may arrive out of order.

15
Why Internet Uses Datagram (Best Effort)
🔵 Definition

The Internet uses Datagram + Best Effort service because it keeps the network layer simple, works over any technology, and moves reliability to end systems (TCP).

🔴 5 Reasons
  • 1. End systems are smart → handle reliability (TCP), ordering, congestion
  • 2. Minimal guarantees → minimal requirements on routers
  • 3. Works over ANY link: Ethernet, WiFi, Satellite, Fiber, Radio
  • 4. Easier to interconnect very different network technologies
  • 5. Easy to add new services: just attach host + define new app protocol
⚡

Quick Summary: Internet = Datagram + Best Effort. Reliability handled by TCP. Works over any link type. Easy to scale.

17
Router Component Functions (Detailed)
🔴 Functions of Each Component
  • Input Port: Physical layer (receives bits) + Link layer (decapsulation) + Lookup (finds output port). Control packets → forwarded to Routing Processor.
  • Switching Fabric: Connects all input ports to all output ports. A network INSIDE the router!
  • Output Port: Stores packets from switching fabric → transmits on outgoing link with link-layer encapsulation
  • Routing Processor: Executes routing protocols (OSPF, BGP, RIP) + maintains routing tables + computes forwarding table + network management
⚡

Quick Summary: Input = receive + lookup. Fabric = internal transfer. Output = buffer + transmit. Routing Processor = routing protocols + forwarding table.

Forwarding Plane vs Control Plane
🔵 Definition

The Forwarding Plane (hardware) moves packets at nanosecond speed. The Control Plane (software) makes routing decisions at millisecond speed.

📐 Numerical: Why Hardware is Needed

Given: 10 Gbps input link, 64-byte IP datagram

Packet size = 64 × 8 = 512 bits Link speed = 10 × 10⁹ bps Time available = 512 / 10,000,000,000 = 51.2 nanoseconds ⚡ → WAY too fast for software → MUST be hardware
PropertyForwarding PlaneControl Plane
SpeedNanosecondsMilliseconds
ImplementationHardwareSoftware
FunctionMove packetsDecide routes
⚡

Quick Summary: At 10Gbps, only 51.2ns per packet → needs hardware. Forwarding Plane = hardware (ns). Control Plane = software (ms).

20
Shadow Copy, TCAM & Advanced Lookup
🔵 Definition

A shadow copy of the forwarding table is stored at each input port for local fast lookup. TCAM = ultra-fast memory that finds longest prefix matches in one clock cycle.

🔴 Key Points
  • Shadow Copy = Copy of forwarding table at each input port → no central bottleneck
  • Called Distributed Forwarding
  • At Gigabit speeds → lookup must happen in nanoseconds
  • DRAM → fast but not fastest
  • SRAM → faster than DRAM
  • TCAM → searches ALL entries simultaneously in ONE clock cycle. Supports 0, 1, or "don't care (X)" → perfect for prefix matching
🟢 Analogy

Shadow copy = Each employee has own phone directory. TCAM = Magic phonebook that finds a name INSTANTLY — all pages searched at once.

⚡

Quick Summary: Shadow copy at each input → avoids bottleneck. TCAM = fastest memory, searches all entries in 1 cycle.

21
Blocking at Input Port
🔵 Definition

A packet at an input port may be temporarily blocked from entering the switching fabric if it is currently busy. The packet is queued at the input port.

🔴 Key Points
  • If switching fabric busy → packet queued at input port
  • Scheduled to cross fabric at later time
  • Other tasks: (1) Physical + link layer processing (2) Check version, checksum, TTL → rewrite (3) Update network management counters
⚡

Quick Summary: Fabric busy → packet queued at input port. 3 additional tasks: physical/link, check/rewrite TTL+checksum, update counters.

Switching via Crossbar
🔵 Definition

A Crossbar Switch is an interconnection network of 2N buses connecting N input ports to N output ports. Multiple packets can be forwarded simultaneously in parallel.

📊 Crossbar Grid Explanation
1

N input ports have horizontal buses, N output ports have vertical buses

2

At each intersection = a crosspoint that can open/close

3

To send A→X: close crosspoint at (A row, X column). Simultaneously B→Y and C→Z → parallel forwarding ⚡

4

If A and B both want output X → one must wait (output conflict)

🔴 Key Points
  • Crossbar = 2N buses connecting N inputs to N outputs
  • Multiple packets forwarded in parallel if different output ports
  • Fastest — used in modern high-performance routers
⚡

Quick Summary: Crossbar = 2N buses, N×N grid. Parallel forwarding possible. Fastest. Conflict only if two packets target same output port.

27
Queuing at Input and Output Ports
🔵 Definition

Packet Queuing occurs when packets arrive faster than they can be processed. If queues overflow → packet loss.

🔴 Key Points
  • R_line = link rate | R_switch = switching fabric rate
  • If R_switch = N × R_line → negligible queuing at input ports
  • If all N packets go to SAME output → output bottleneck → memory exhaust → packet loss
  • Output port can transmit only 1 packet at a time
📐 Concept
R_switch = 3 × R_line ✅ (no input port queuing) BUT output port sends 1 packet/unit time → 3 packets arrive → 2 must queue at OUTPUT → If continues → buffer fills → packet loss ❌
⚡

Quick Summary: Output queue when multiple packets target same output. R_switch = N × R_line → no input queue. Memory full → packet loss.

28
Packet Scheduler, WFQ & RED
🔵 Definition

A Packet Scheduler decides which queued packet to send next. WFQ shares bandwidth fairly. RED proactively drops packets before buffer overflows.

🔴 3 Scheduling Algorithms
  • 1. FCFS/FIFO – Transmit in exact arrival order. No priority.
  • 2. WFQ – Different connections get weighted share of bandwidth. High-weight = more bandwidth. Good for QoS.
  • 3. RED – Probabilistically marks/drops packets BEFORE buffer is full. Signals congestion early. Prevents sudden large drops.
🟢 Analogy

FCFS = Bank queue. WFQ = VIP counter (priority customers faster). RED = Restaurant stopping reservations before fully booked.

⚡

Quick Summary: FCFS = simple fair. WFQ = weighted bandwidth (QoS). RED = proactive drop before buffer full. All manage output queues.

29
HOL (Head-of-the-Line) Blocking
🔵 Definition

HOL Blocking occurs when a packet at the front of a queue is waiting for a busy output port, blocking packets behind it — even if THEIR output ports are free.

📊 HOL Blocking Scenario
1

Dark blue packets from Input 1 AND Input 3 both want Upper output port (contention)

2

Only ONE dark blue can proceed → other waits at head of queue

3

Light blue packet is BEHIND the waiting dark blue in queue

4

Light blue wants Lower output port (which is FREE!) but CANNOT move → HOL Blocked ❌

🔴 Key Points
  • HOL Blocking = packet at HEAD blocks packets behind it
  • Occurs at input ports
  • Queue grows to unbounded length
  • Can reduce throughput to only 58.6% of maximum
  • Solution: Virtual Output Queues (VOQ)
🟢 Analogy

Supermarket: Customer ahead has price check issue (blocked). You have 2 items and checkout is free — but you CANNOT move past blocked customer. = HOL blocking.

⚡

Quick Summary: HOL = head blocks others even if their output is free. Reduces throughput to ~58.6%. Fix = VOQ (Virtual Output Queues).

Routing Plane & Software Defined Networking (SDN)
🔵 Definition

The Routing Plane (Control Plane) runs routing protocols and computes forwarding tables. SDN separates the control plane (software) from the data plane (hardware), allowing external controllers to manage routing.

🔴 Key Points
  • Traditional: Control plane runs inside each router (decentralized)
  • SDN: Control plane can be external to the router
  • A well-defined API allows internal and external parts to communicate
  • SDN = separates software control plane from hardware data plane
  • External SDN Controller pushes forwarding table entries to hardware routers
  • SDN heavily used in Cloud Computing and Data Centers
PropertyTraditional RoutingSDN
Control PlaneInside each routerExternal controller
FlexibilityLowHigh
Use CaseTraditional networksCloud, Data Centers
🟢 Analogy

Traditional = Each car has own GPS. SDN = Central traffic management system tells ALL cars which routes to take.

⚡

Quick Summary: SDN = external controller manages multiple routers via API. Control plane = software. Data plane = hardware. Used in cloud/data centers.
